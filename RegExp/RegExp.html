# 正则表达式

# 创建

正则表达式是用于匹配字符串中字符组合的模式。

在 JavaScript 中，正则表达式是一个标准内置对象（ RegExp对象）。

有种方法可以创建一个 RegExp 对象：一种是字面量，另一种是构造函数。

  • 字面量
    var re = /\w+/;

  • 构造函数

    // 参数一可以是常规元字符字符串，也可以是正则表达式字面量
new RegExp('ab+c', 'i'); // 首个参数为字符串模式的构造函数
new RegExp(/ab+c/, 'i'); // 首个参数为常规字面量的构造函数

使用字面量创建的正则表达式即使重复执行代码也只会被编译一次，使用构造函数创建的正则表达式在代码每次执行期间都会被重新编译。因此，如果保存的是常量，推荐使用字面量；如果保存的是会在未来发生变化的值，推荐使用构造函数。

正则表达式可以被用于 RegExp 的 exec 和 test 方法，以及 String 的 match、matchAll、replace、search 和 split 方法。

# 使用

正则表达式通常由两部分组成：元字符 + 修饰符（把这些元字符和修饰符全背下来，正则就通关了）。元字符是用来创造正则规则的，修饰符是用来在规则外添加额外规则的。

TIP

正常文本字符也可以作为正则表达式的一部分，用来匹配其自身

常用的元字符：

  1. 量词元字符：匹配前一个表达式出现的次数

    * 零到多次

    + 一到多次

    ? 零次或者一次

    {n} 出现n次 /\d{5}/代表一个数字字符出现了5次，即是一个5位数

    {n,} 出现n到多次

    {n,m} 出现n到m次（开区间，n和m都可以是0次）

  2. 特殊元字符：单个或者组合在一些代表特殊的含义

    1. 单个

      \ 转义字符

      两个作用，可以把普通字符转译成带特殊含义的，也可以把特殊含义字符转义成普通字符

      如果你想将字符串传递给 RegExp 构造函数，不要忘记在字符串字面量中反斜杠是转义字符。所以为了在模式中添加一个反斜杠，你需要在字符串字面量中转义它。为了通过字符串字面量给 RegExp 构造函数创建包含反斜杠的表达式，你需要在字符串级别和正则表达式级别都对它进行转义。

      . 除\n（换行符）以外的任何字符

      .+ 代表除\n以外的一到多个字符

      ^ 以哪一个元字符作为开始

      $ 以哪一个元字符作为结束

    2. 组合

      \n 换行符

      \d 0~9之间的一个数字

      \D 非0~9之间的一个数字（大写和小写的意思是相反的）

      \w 数字、字母、下换线中的任意一个字符

      \s 一个空白字符（包含空格、制表符、换页符等）

      \t 一个制表符（一个TAB键：四个空格）

      \b 匹配一个单词的边界

      x|y x或者y中的一个字符（可以写多个）

      [xyz] x或者y或者z中的任意一个字符

      [^xy] 除了x/y以外的任意字符

      [a-z] a-z这个范围中的任意字符 [0-9a-zA-Z_]===\w

      [^a-z] 非a-z这个范围中的任意字符

      () 分组符号

      (?:) 只匹配不捕获

      (?=) 正向预查

      (?!) 负向预查

    3. 普通元字符：代表本身含义的

      /xusong/ 匹配的就是字符串"xusong"

常用的修饰符

  1. i ignoreCase 忽略大小写匹配
  2. m multiline 多行匹配
  3. g global 全局匹配

# 属性

# RegExp.lastIndex

lastIndex 是正则表达式的一个可读可写的整型属性，用来指定下一次匹配的起始索引。

只有正则表达式使用了表示全局检索的 "g" 或者粘性检索的 "y" 标志时，该属性才会起作用。此时应用下面的规则：

  • 如果 lastIndex 大于字符串的长度，则 regexp.test 和 regexp.exec 将会匹配失败，然后 lastIndex 被设置为 0。
  • 如果 lastIndex 等于或小于字符串的长度，则该正则表达式匹配从 lastIndex 位置开始的字符串。
  • 如果 regexp.test 和 regexp.exec 匹配成功，lastIndex 会被设置为紧随最近一次成功匹配的下一个位置。
  • 如果 regexp.test 和 regexp.exec 匹配失败，lastIndex 会被设置为 0

# RegExp.prototype.multiline

multiline 属性表明正则表达式是否使用了 "m" （多行匹配）标志。multiline 是正则表达式实例的一个只读属性。multiline 是一个布尔对象，如果使用了 "m" 标志，则返回 true；否则，返回 false。"m" 标志意味着一个多行输入字符串被看作多行。例如，使用 "m"，"^" 和 "$" 将会从只匹配正则字符串的开头或结尾，变为匹配字符串中任一行的开头或结尾。

# 方法

# RegExp.prototype.exec()

exec() 方法位于 RegExp 对象的原型对象上，可供所有的正则表达式实例使用。

exec() 方法在一个指定字符串中执行一个搜索匹配。返回一个结果数组或 null。

const regex1 = RegExp('foo*', 'g');
const str1 = 'table football, foosball';
let array1;

while ((array1 = regex1.exec(str1)) !== null) {
  console.log(`Found ${array1[0]}. Next starts at ${regex1.lastIndex}.`);
  // Expected output: "Found foo. Next starts at 9."
  // Expected output: "Found foo. Next starts at 19."
}

# RegExp.prototype.test()

test() 方法执行一个检索，用来查看正则表达式与指定的字符串是否匹配。返回 true 或 false。

如果正则表达式设置了全局标志，test() 的执行会改变正则表达式 lastIndex属性。连续的执行test()方法，后续的执行将会从 lastIndex 处开始匹配字符串，(exec() 同样改变正则本身的lastIndex 属性值).

var regex = /foo/g;

// regex.lastIndex is at 0
regex.test('foo'); // true

// regex.lastIndex is now at 3
regex.test('foo'); // false

# String.prototype.match()

match() 方法检索返回一个字符串匹配正则表达式的结果。

参数是一个正则表达式，如果使用 g 标志，则将返回与完整正则表达式匹配的所有结果，但不会返回捕获组。如果未使用 g 标志，则仅返回第一个完整匹配及其相关的捕获组（Array）。在这种情况下，返回的项目将具有如下所述的其他属性。

# String.prototype.matchAll()

matchAll() 方法返回一个包含所有匹配正则表达式的结果及分组捕获组的迭代器。

参数是一个正则表达式，RegExp必须是设置了全局模式g的形式，否则会抛出异常TypeError。

# 比较

  • 如果你需要知道一个字符串是否与一个正则表达式匹配 RegExp，可使用 RegExp.test() 。
  • 如果你只是需要第一个匹配结果，你也可以使用 RegExp.exec() 。
  • 如果你想要获得捕获组，并且设置了全局标志，你需要用 RegExp.exec()或者 String.prototype.matchAll()

# String.prototype.replace()

replace() 方法返回一个由替换值（replacement）替换部分或所有的模式（pattern）匹配项后的新字符串。模式可以是一个字符串或者一个正则表达式，替换值可以是一个字符串或者一个每次匹配都要调用的回调函数。如果pattern是字符串，则仅替换第一个匹配项。

参数一是一个正则表达式对象或者要背替换的字符串，参数二是用来替换的新字符串或者一个回调函数。

# String.prototype.search()

search() 方法执行正则表达式和String对象之间的一个搜索匹配。

参数是一个正则表达式对象，如果匹配成功，则 search() 返回正则表达式在字符串中首次匹配项的索引；否则，返回 -1。

# 字符组（也叫字符集）

正则表达式中，方括号是特殊元字符，不参与匹配。

通过方括号可以指定字符组，写法是在方括号中列出想要匹配的字符，这些字符和方括号整体就构成一个字符组。

字符组有以下几个特点：

  1. 字符组中出现的字符一般都代表其本身含义（相对于元字符进行理解）
  2. 字符组中不存在多位数
  3. 字符组中的字符排列顺序不影响字符组的功能，出现重复字符也不影响

# 分组

使用小括号 () 可以对正则表达式进行分组，左括号(表示捕获分组的起始符，右括号)表示捕获分组的结束符。分组的目的之一在于匹配（重复出现的）多个不同字符（单个字符直接加量词就可以了），另一个目的是提高处理的优先级。

定义分组有三种形式：

  • (exp) ：把括号内的正则作为一个分组，系统自动分配组号，可以通过分组号引用该分组；
  • (?P<name>exp)：定义一个命名分组，分组的正则是exp，系统为该分组分配分组号，可以通过分组名或分组号引用该分组；
  • (?:exp)：定义一个不捕获分组，该分组只在当前位置匹配文本，在该分组之后，无法引用该分组，因为该分组没有分组名，没有分组号，也不会占用分组编号；

分组编号默认从1开始递增，整个正则表达式是一个隐含的全局分组，编号是0。

# 捕获

捕获是指使用小括号指定一个子表达式后，子表达式匹配的文本（即匹配的内容）可以在其他子表达式中重复使用。

# 后向引用

可以通过名称和编号来引用前面已经出现的分组，获取该分组捕获到的文本。

正则表达式中，可以通过分组名或分组号来引用：

  • (?P=name)：引用名称为name的分组
  • \n：使用分组的编号来引用分组，n代表分组在正则表达式中的组号。

# 断言

断言也称零宽度断言（匹配的是个没有宽度的东西，而不是空格），用来标记边界，即匹配字符串中的位置，而不是匹配字符。

# 行的起始与结束

^（脱字符）会匹配行或者字符串的起始位置，$会匹配行或者字符串的结束位置。这两个东西也叫做锚位符（anchor）。有时也用来匹配整个文档的起始和结束位置。

# 单词边界与非单词边界

\b 用来匹配单词边界，\B 用来匹配非单词边界，也都是零宽度断言。
