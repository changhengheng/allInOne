# 初识vue

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>1.vue初体验</title>
    <!-- 1.引入Vue，暴露一个 Vue 构造函数 -->
    <script type="text/javascript" src="../js/vue.js"></script>
  </head>
  <body>
    <!-- 2.准备好一个容器-->
    <div id="app">
      <h1>{{str}}</h1>
    </div>
  </body>

  <script type="text/javascript">
    // 禁用生产提示
    Vue.config.productionTip = false;
    // 实例化调用Vue构造函数
    new Vue({
      el: "#app", // 绑定视图容器，值为 css 选择器
      data() {
        // vue中的状态存放的位置，值暂时为对象
        return {
          str: "hello world!",
        };
      },
    });
  </script>
</html>

el和data配置项

el 相当于 element，绑定视图容器，值为 css 选择器

data 是 Vue 中的状态数据存放的位置，值暂时为对象

TIP

data里可以放函数，但不建议这么做。

# vue 容器 和 实例对象 的关系

vue 容器和 实例对象的关系 必须一一对应。 vue 容器 只能对应一个 vue 实例 vue 实例 只能对应一个 vue 容器

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>1.vue初体验</title>
    <script type="text/javascript" src="../js/vue.js"></script>
  </head>
  <body>
    <!-- 容器1 -->
    <div class="app">
      <h1>{{str}}</h1>
    </div>
    <!-- 容器2 -->
    <div id="app">
      <h1>{{str}}</h1>
    </div>
  </body>

  <script type="text/javascript">
    Vue.config.productionTip = false;
    // 实例1
    new Vue({
      el: ".app", 
      data() {
        return {
          str: "hello world!",
        };
      },
    });
    // 实例2
    new Vue({
      el: "#app", 
      data() {
        return {
          str: "你好，世界!",
        };
      },
    });
  </script>
</html>

# vue表达式 {{}}——插值语法

插值语法中可以写的东西：

  1. 数据类型：
    • 展示： string 、number、boolean、object、array
    • 不展示：undefined、null
    • 有返回值的方法，且返回值为 string 、number、boolean、object、array
  2. 不允许写语句
    • if……else
    • switch
    • 循环 for、while 、 do……while、for in 、for of、forEach

# vue 模板语法

  1. 插值语法 {{}} :

    能解析标签体中的文本的内容

  2. 指令语法 v-：

    Vue提供的一种标签的动态属性

插值语法可以展示 vm 身上的所有的属性，包括原型对象上的属性 指令语法可以解析 vm 身上的所有的属性，包括原型对象上的属性

  <h1>{{_uid}}</h1>
  <h1>{{$isServer}}</h1>
  <!-- 审查元素查看这两个属性是否被解析 -->
  <a v-bind:href="url" v-show="$isServer" v-bind:aa="_uid">百度一下</a>

_uid 是vm身上的属性，$isServer 是vm的原型对象上的属性。在以上两种情况下都可以解析出来。

# v-bind 指令

作用：动态解析属性值

全写：

<!-- 看这个节点，添加的属性被解析了 -->
<h1 v-bind:aaa="_data"></h1>

v-bind 指令会将属性值作为动态数据区解析

简写形式：

<h1 :aaa="_data"></h1>

单向数据绑定：只能通过数据驱动视图，视图不能驱动数据

数据可以渲染到页面中去，但是页面中视图的改变不能引起对应数据的改变

<div id="app">
  <!-- 全写 -->
  <a v-bind:href="url">百度一下</a>
  <!-- 简写 -->
  <img :src="img">
</div>
<script>
  new Vue({
    el: "#app",
    data: {
      url:"https://www.baidu.com",
      img:"./img/1.jpg"
    },
  });
</script>

TIP

_data是vm实例身上的data配置项

# v-model

双向数据绑定：数据可以驱动视图，视图也可以驱动数据。

注意点：v-model 指令只能使用在以下输入类的标签中：

  • input
  • select
  • textarea

v-model 指令 会将data中的数据存入到 input 的 value 属性中

<input type="text"  v-model:value="str"  />

使用：

<div id="app">
    <!-- 全写 -->
    <input v-model:value="str" type="text" />
    <!-- 简写：省略了value -->
    <input v-model="str1" type="text" />
</div>
<script>
    new Vue({
        el: "#app",
        data: {
            str: "hello world!!!",
            str1: "哈哈哈哈哈",
        },
    });
</script>

# v-model修饰符

  1. .lazy - 取代 input 监听而使用 change 监听
  2. .number - 输入字符串转为有效的数字
  3. .trim - 忽略首尾两端的空格
<!--全写-->
<input v-model:value.lazy="str" type="text" />
<!--简写-->
<input v-model.lazy="str1" type="text" />

<!--链式调用-->
<div id="app">
    <input v-model.number.lazy.trim="str" type="text" />
</div>
<script>
    new Vue({
        el: "#app",
        data: {
            str: "      123        ",
        },
    });
</script>

TIP

v-bind可以添加在任何属性上，v-mode只能添加在输入类表单上

v-model练习：收集单选、多选、下拉列表的数据

v-model 收集数据时，单选、多选和下拉列表没有值，需要通过 value 指定值，最终收集到的数据就是指定的 value 属性的值。这是原生js的问题，本来就没有值。单选、多选和下拉列表都是不能直接输入的。

TIP

区分：给用户看的数据&&开发人员需要收集的真正的数据

输入框收集的数据都是string类型

# MVVM

M model【模型】 数据 状态 实例中的 data V view 【视图】 html + css 模板 VM 虚拟DOM 实例对象

vue 就是 MVVM 思想的实现。

MVC 思想 M model【模型】 V view 【视图】 C controler 控制器

TIP

VM是虚拟DOM，使用diff算法优化视图和数据之间的联系，效率更高，不用操作真实DOM

MVC是一种思想，用来指导所有的语言：springmvc，前端只留在视图层（MVC的M）

java是用来写后台逻辑的，用来联系前端和数据库，数据库语言是sql

# Object.defineProperty

作用:给对象添加一个高度定制的属性

let num = 30;

let obj = {
  uname: "张三",
  sex: "男"
}

Object.defineProperty(obj, "age", {
  value:18,   //默认值
  enumerable:true,   //  enumerable 设置值是否可参与枚举【遍历】，默认值为false
  configurable:true,  //  configurable设置值是否可删除，默认值为false
  writable:true  // writable   设置值是否可修改，默认值为false 

  //  当有人读取 age 的值时，就会自动调用 get 方法，get的返回值就是 age 的值。
  get() {
    console.log("get")
    return num;
  },
  // 当有人修改 age 的值时，就会自动调用 set 方法 
  set(value) {
    console.log("set");
    // console.log(value);
    num = value;
  }
});

# 数据代理

概念：一个对象代理对另外一个对象属性的操作：【读/写】。

# js 数据代理

let a = 1;
let obj = {
    name: "sssss",
};
Object.defineProperty(obj, "age", {
  get() {
    console.log("get方法");
    // 读值时返回该返回值
    return a;
  },
  // 新设置的值
  set(newVal) {
    console.log("set方法", newVal);
    // 赋值给变量a
    a = newVal
  },
});
// 读age得到的是a，设置age会设置给a；设置a，读age会得到，读a，读的是age
// 代理完之后，代理对象身上就有这个属性了

# vue中的数据代理

  1. 将 Vue 配置对象的 data 作为 vm 实例对象的 _data;
  2. 循环遍历【枚举】 _data 中的所有的属性
  3. 为遍历的所有的属性 进行 Object.defineProperty(vm,遍历的属性，{});
  4. 将数据代理到 vm 实例对象上 【数据代理： 一个对象(vm)代理对另外一个对象(_data)属性(_data存放的所有的数据)的操作【读/写】。

优点： 简化程序员的模板代码 不代理： <h1>{{_data.uname}}</h1> 代理后： <h1>{{uname}}</h1>

<body>
  <!-- 准备好一个容器-->
  <div id="app">
    <h1>{{uname}}</h1>
<h1>{{age}}</h1>
</div>
</body>
<script type="text/javascript">
  //数据代理： 一个对象(vm)代理对另外一个对象(_data)属性(_data存放的所有的数据)的操作【读/写】。
  let vm = new Vue({
    el:"#app",
    data:{
      uname:"张三",
      age:18
    }
  })
// console.log(vm._data === data );
console.log(vm)
/*  
  Vue 内部执行原理——伪代码
  循环遍历 _data.,开始代理
  Object.defineProperty(vm,"uname",{
    get(){
      return _data.uname;
    },
    set(value){
      _data.uname = value;
    }
  })

  Object.defineProperty(vm,"age",{
    get(){
      return _data.age;
    },
    set(value){
      _data.age = value;
    }
  })
.... */
</script>

TIP

Vue是Vue.js暴露出来的一个构造函数，vm是该构造函数的实例对象，就是MVVM中的VM，用来提高视图和数据之间联系的效率

vm实例对象的原型链上的东西都能用模板语法展示：可以用插值语法展示，也可以用 v-bind 解析

data配置项里的this指向实例对象vm
