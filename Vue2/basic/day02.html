# 原生JS事件绑定

原生JS事件绑定有三种方式：

HTML

<div id="btn">test</div>

JS

document.getElementById("btn").onclick = function(){
  alert(1234567890);
}

HTML

<div id="btn">test</div>

JS

document.getElementById("btn").addEventListener("click",function(){
  alert(1234567890);
})

HTML

<div id="btn" onclick="demo()">test</div>

JS

function demo(){
  alert(1234567890);
}

Vue中只能使用行内事件绑定。因为和React一样，Vue中vm使用虚拟DOM优化了视图和数据交互的效率，而前两种方式都是基于真实DOM的，因此不能使用。

# methods配置项

methods中一般只放置普通方法，即事件调用的方法。

methods里的方法不会走代理，而是会直接挂到vm身上，data里放的东西都会走代理。 所以普通方法也可以放data里，但是效率不高（面试题），我们也不会这么做，因为通常我们不会调一个方法的set，而只会调它的get，而data里的东西都会走代理（都会添加get和set），所以普通方法放data中时效率低。

# 数据响应

  1. vue 是双向数据绑定的框架 Object.defineProperty
  2. react 是单向数据绑定的框架

# Vue中的事件绑定

Vue使用 v-on指令为虚拟dom绑定JS事件：

<div id="app">
  <button v-on:click="demo">{{str}}</button>
</div>

TIP

v-on:事件类型

vue 事件回调的函数啊必须是 普通函数，因为 this 指向 vm 实例对象，如果使用箭头函数，会丢失this

<h1 @click="demo" >宁愿带着浑身伤痛勇敢站起 让泪水洒满尘埃与苍穹之间</h1>

TIP

@:事件类型

# 事件修饰符

.stop 阻止冒泡

.self 只当事件是从侦听器绑定的元素本身触发时才触发回调。

.prevent 调用 event.preventDefault()。 阻止默认行为

TIP

默认行为总结：

  1. a链接跳转
  2. submit提交
  3. 右键菜单
  4. 复制粘贴

.once - 只触发一次回调。触发回调后，会自动解绑当前的事件。

TIP

.once本质上会在触发一次之后解绑当前事件（不是清理回调）

键盘修饰符

.{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调。

<!-- 键修饰符，键别名 -->
<input @keyup.enter="onEnter">
<!-- 键修饰符，键代码 -->
<input @keyup.13="onEnter">

TIP

keyAlias指按键别名，比如enter、q、w

keyCode指键码，比如enter是13

鼠标按键修饰符

  • .left - (2.2.0) 只当点击鼠标左键时触发。
  • .right - (2.2.0) 只当点击鼠标右键时触发。
  • .middle - (2.2.0) 只当点击鼠标中键时触发。

链式调用

<div class="one" v-on:click="one">
  <div class="two"  v-on:click="two">
    <div class="three"  v-on:click="three">
      <a href="https://www.baidu.com" v-on:click.prevent.stop.once="demo3">百度</a>
    </div>
  </div>
</div>

TIP

所有修饰符都可以链式调用

# 计算属性

风格指南open in new window：为什么要使用计算属性？

使用场景：想在DOM中使用一个当前data中没有、但可以通过已有data属性计算出来的属性时。

为什么不使用插值语法和methods？

因为：

  1. 不推荐在插值表达式中写复杂逻辑

  2. 不推荐在methods中写复杂逻辑（只放普通方法）

TIP

methods会引起视图多次渲染（参考下面methods中方法的特点）

插值语法中的表达式不推荐太臃肿

下面是一个例子：

表达式写法

<body>
  <!-- 准备好一个容器-->
  <div id="app">
    姓：<input type="text" v-model="firstName" /><br><br>
    名：<input type="text" v-model="lastName" /><br><br>
    <h1>全名： {{firstName[0].toUpperCase()+firstName.slice(1)}}-  {{lastName[0].toUpperCase()+lastName.slice(1)}}</h1>
  </div>
</body>

<script>
  new Vue({
    el:"#app",
    data:{
      firstName:"zhang",
      lastName:"san"
    }
  })
</script>

事件写法：

<body>
  <div id="app">
  <h1>{{age}}</h1>
  <input type="text" v-model="age">

  <hr>

  姓：<input type="text" v-model="firstName" /><br><br>
  名：<input type="text" v-model="lastName" /><br><br>
  <h1>全名：{{fullName()}}</h1>
  </div>
</body>

<script>
  let vm = new Vue({
    el:"#app",
    data:{
      age:18,a
      firstName:"zhang",
      lastName:"san"
    },
    methods:{
      fullName(){
        console.log("我是fullName，我被执行了")
        // console.log(this);
        //解构赋值
        let {firstName,lastName} = this;
        return firstName[0].toUpperCase()+firstName.slice(1)  +"-"+  lastName[0].toUpperCase()+lastName.slice(1);
      }
    }
  })
</script>

# methods深入

methods中方法的特点：

  1. 不调用不执行，只要调用立即执行
  2. 调用几次执行几次，没有缓存

Vue执行流程：初始化渲染一次，数据变化影响视图时会重新渲染。

因此，methods中的方法多次执行时会引起视图多次渲染，造成过高的损耗。

解决方案：computed

# computed计算属性

基本使用

computed是一个配置项，值是一个对象。其中计算的数据也是一个对象，每个数据对象中有 get 和 set 两个方法，读取该值时执行 get 方法，get 方法的返回值就是最终计算的值；设置该值时执行set方法。get和set方法中的this都指向当前实例对象。

特点

computed计算的数据和data数据一样，会被代理到vm身上，因此可以直接在模板中使用，且可以省略this。

全写

<div id="app">
  姓 <input type="text" v-model="firstName" />
  <br />
  名 <input type="text" v-model="lastName" />
  <br />
  姓名：{{fullName}}
  <button @click="change">点击改名</button>
</div>

<script type="text/javascript">
  new Vue({
    el: "#app",
    data() {
      return {
        firstName: "zhang",
        lastName: "san",
      };
    },
    methods: {
      change() {
        this.fullName = "li-si";
      },
    },
    computed: {
      fullName: {
        get() {
          return (
            this.firstName[0].toUpperCase() +
            this.firstName.slice(1) +
            "-" +
            this.lastName[0].toUpperCase() +
            this.lastName.slice(1)
          );
        },
        set(value) {
          console.log("set");
          let [firstName, lastName] = value.split("-");
          this.firstName = firstName;
          this.lastName = lastName;
        },
      },
    },
  });
</script>

# set的使用

计算出来的数据不允许直接修改，直接改会报错。

只能在set中修改，而且set中也不能直接修改计算数据本身，否则set会触发set，会导致代码无限循环执行。应该直接修改依赖数据。

但一般不使用set方法，因为该数据是计算出来的，如果还需要修改，说明计算出了问题。

# 计算属性的简写

函数写法：

<div id="app">
    姓 <input type="text" v-model="firstName" />
    <br />
    名 <input type="text" v-model="lastName" />
    <br />
    姓名：{{fullName}}
    <button @click="change">点击改名</button>
</div>
<script type="text/javascript">
    new Vue({
        el: "#app",
        data() {
            return {
                firstName: "zhang",
                lastName: "san",
            };
        },
        methods: {
            change() {
                this.fullName = "li-si";
            },
        },
        computed: {
            fullName() {
                return (
                    this.firstName[0].toUpperCase() +
                    this.firstName.slice(1) +
                    "-" +
                    this.lastName[0].toUpperCase() +
                    this.lastName.slice(1)
                );
            },
        },
    });
</script>

函数写法等价于对象写法的get。

# data的特点

data数据是响应式的，响应式就是说视图和数据可以相互影响。数据变化会引起视图变化（重新渲染视图），视图变化也会引起数据变化（数据发生对应的改变）。

Vue2响应式原理是通过Object.defineProperty劫持data数据实现的，也就是说Vue内部可以通过对data数据进行监视，得知data数据的变化。因为读取数据会触发getter，设置数据会触发setter。而除此之外的方式引起的数据变化，比如对没有添加get和set的数据直接赋值，Vue是无法监听的，也就无法触发视图更新。比如，直接修改数组元素的值就无法触发视图更新，因为数组元素是被有被代理的。

# 监视属性

Vue通过 Object.defineProperty 监听data数据的原因是为了更新视图，Vue同时也提供了一种让开发者主动监听数据的方式——watch，方便开发者添加自己的逻辑。

基本使用

watch配置项是一个对象，对象中存放监视的属性数据，每一个数据也是一个对象，其中有一些配置项：

  • handler：当监听数据发生变化时执行的回调，默认参数分别是 newVal，oldVal
  • immediate：立即执行一次 handler

watch只能监听vm上存在的数据，比如data、computed的数据。

watch: {
 somedata: {
  handler(newVal, oldVal) {
   console.log(newVal,oldVal);
  },
 },
},

# $watch()

位于vm的原型对象上的方法，也能监听数据。参数一是要监听的数据（位于vm身上的数据），参数二是数据发生变化时触发的回调，也可以是一个配置对象。

基本不用，了解即可。

<body>
  <div id="app">
    <h1>{{num}}</h1>
    <button @click="num++">点击</button>
  </div>
</body>

<script type="text/javascript">
  let vm = new Vue({
    data() {
      return {
        num: 1,
      };
    },
  }).$mount("#app");
  vm.$watch('num',function(newVal,oldVal){
      console.log('num发生变化了');
  })
</script>

# 对象属性的严格写法

对象的属性要么是字符串，要么是symbol。平时为了代码简单，可以不写双引号。而在watch监视属性中，因为watch是一个对象，想要监听一个对象中的属性时，需要使用严格的字符串的写法（因为对象中的属性不在vm上，没有办法直接监听）。

watch: {
 "user.name": {
  handler(newVal, oldVal) {
   console.log(newVal,oldVal);
  },
 },
},

# 深度监视

如果要监视的对象的属性非常多，上面的写法会非常繁琐，Vue提供了deep配置项，可以同时监视对象数据的所有属性及对象本身的变化（在变化时触发handler回调）。

vm.$watch("somedata", {
  handler(newVal) {
      console.log(newVal);
  },
  immediate:true,
  deep:true
});

# 简写形式

函数形式，相当于handler：

watch: {
    somedata(newVal,oldVal){
  console.log (newVal,oldVal);
    }
},

# computed和watch总结

# computed 计算属性

  1. DOM 使用的数据在data中不存在, 是通过已有的data 中的存在的数据计算而来的，才需要使用计算属性。 【如果一个属性是由其他属性计算而来的，这个属性依赖其他属性，是一个【多对一】或者【一对一】，一般用computed】
  2. 写法方法 (1) 当计算属性为 对象时：当有人读取 计算属性的值时，会自动调用内部的get方法。有人更新计算属性的值时，会自动调用set方法。set接受到的值，就是要更新的数据。不能去直接更新 计算属性的值，应该去更新 依赖的数据。 (2) 当计算属性为 函数时，此时，函数 就是 get 方法。
  3. 计算属性 会自动代理到 vm 实例对象上。
  4. 计算属性是有缓存的，只有当依赖的数据发生变化时，才会重新执行计算，否则，走缓存。
  5. 计算属性 不支持异步，当computed内有异步操作时无效，无法监听数据的变化
  6. computed 能完成的 ，watch都能完成。但是，watch能做的，computed不一定能做。

# watch 监听属性 监视属性

  1. 当数据发生变化，我们写入自己的逻辑时，需要主动使用 watch 监听属性。
  2. 只能监听 vm 上 存在的数据【data、 computed】
  3. 【handler】 当 要监听的数据发生变化时，执行回调 ，回调函数接受两个参数。第一个为新数据，第二个为老数据。 【immediate:true】 立即监视 【deep:true】 深度监视
  4. watch 每次只能监听一个数据，根据监听的数据可以更新一个或者多个数据。【一对多或者一对一】
  5. watch 没有缓存。
  6. watch 支持异步。
  7. computed 能完成的 ，watch都能完成。但是，watch能做的，computed不一定能做。

# wacth 和 computed 总结对比

  1. wacth 监听已有数据。 computed 通过已有数据计算新的数据。
  2. wacth 【一对多或者一对一】. computed 【多对一】或者【一对一】
  3. wacth 没有缓存。 computed 有缓存
  4. watch 支持异步。 computed 不支持异步
  5. computed 能完成的 ，watch都能完成。但是，watch能做的，computed不一定能做。

          },

            let usernameReg = /^[\u4e00-\u9fa5]{0,}$/;
            console.log(1);
            return usernameReg.test(this.username) || this.username == "";
          },
          isEmpty() {
            // 提交时输入为空提示不能输入空串

              this.account === "" ||
              this.password === "" ||
              this.phone === "" ||
