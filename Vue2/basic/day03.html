# 动态类

换成id也是一样的

<style>
    .one{
        width: 300px;
        height: 300px;
        border: 1px solid red;
    }

    .two{
        background: hotpink;
    }

    .a{
        background: hotpink;
    }

    .b{
        border-radius: 30%;
    }

    .c{
        text-align: center;
        font-size: 20px;
    }
</style>

字符串：样式不固定，个数也不固定。

<body>
    <div id="app">
        <h1 class="one" :class="str"  >学习vue动态绑定class样式</h1>
        <button @click=" str = '' ">点击切换</button>
    </div>
</body>

<script>
    new Vue({
        el:"#app",
        data:{
            str:"two a c"
        }
    })
</script>

对象： 样式固定，个数也固定。不知道用不用。

<body>
    <div id="app">
        <h1 class="one" v-bind:class="obj" >学习vue动态绑定class样式</h1>
        <button @click="obj.a = !obj.a">切换a</button>
        <button @click="obj.b = !obj.b">切换b</button>
        <button @click="obj.c = !obj.c">切换c</button>
    </div>
</body>

<script>
    new Vue({
        el:"#app",
        data:{
            obj:{
                a:true,
                b:true,
                c:true
            }
        }
    })
</script>

数组： 样式不固定，个数也不固定。

<body>
    <div id="app">
        <h1 class="one" :class="arr"  >学习vue动态绑定class样式</h1>
        <input type="text" v-model="str"  v-on:keyup.enter="getValue" />
    </div>
</body>

<script>
    new Vue({
        el:"#app",
        data:{
            arr:[],
            str:""
        },
        methods:{
            getValue(){
                this.arr.push(this.str);
            }
        }
    })
</script>

# v-show

代码加载了，只是不显示。

v-show和v-if能实现的，插值语法通过判断也能做，但是Vue官方不推荐在插值语法中写太多逻辑。

作用： 切换 dom 的显示隐藏

本质：v-show 的值为 false,则动态添加 display:none 来隐藏。

使用场景： 频繁切换dom 使用 v-show

# v-if

作用： 切换 dom 的显示

本质： v-if 的值为 false，则不加载dom结构。

使用场景： 当不需要展示的dom要销毁掉，用 v-if。不频繁切换dom。

# v-for

遍历array、object、number、string

array 输出数组中的每一个值

object 除了key也有下标，遍历时有三个默认参数，分别是：value、key、index

number 遍历一个number时，从1开始遍历到该number值为止

string 遍历一个string时，会输出字符串中的每一个字符

除了number，遍历其他类型时，v-for有两个默认参数，分别是item、index

TIP

v-for还可以循环undefined和null，不会报错，但是不会输出任何内容。

# 其他指令

# v-text

  • 动态替换标签体内容。如果要更新部分的 textContent，需要使用 插值。
<body>
    <div id="app">
        <!-- text会替换掉标签体的内容 -->
        <p v-text="text">hahahaah</p>
        <!-- text1会替换掉text -->
        <p v-text="text1">{{text1}}{{text}}</p>
    </div>
</body>

<script type="text/javascript">
new Vue({
    data() {
        return {
            text: '中午吃什么',
            text1:"事实上"
        };
    },
}).$mount("#app");
</script>

# v-html

  • 更新元素的 innerHTML。注意：内容按普通 HTML 插入 - 不会作为 Vue 模板进行编译。
  • 在网站上动态渲染任意 HTML 是非常危险的，因为容易导致 XSS 攻击。只在可信内容上使用 v-html，永不用在用户提交的内容上。
  • 会替换标签体的内容
  <body>
    <div id="app">
      <p v-html="html">hahahaah</p>
      </p>
    </div>
  </body>

  <script type="text/javascript">
    new Vue({
      data() {
        return {
          html:"<h1>随身水事实上</h1>"
        };
      },
    }).$mount("#app");
  </script>

# v-pre

  • 不需要表达式
  • 跳过这个元素和它的子元素的编译过程。
<body>
    <div id="app">
        <!-- 不会被Vue编译，而会直接渲染 -->
        <p v-pre>{{text}}</p>
        </p>
    </div>
</body>

<script type="text/javascript">
    new Vue({
        data() {
            return {
                text: '中午吃什么',
            };
        },
    }).$mount("#app");
</script>

# v-cloak

  • 这个指令会保持在元素上直到关联实例结束编译。
  • 不需要表达式
  • 配合CSS样式一起使用
<style>
    /* 指令存在时关联的CSS样式 */
    [v-cloak] {
    display: none;
    }
</style>
<body>
    <div id="app">
        <!-- 使用指令关联DOM元素，vm结束编译前不渲染 -->
        <p v-cloak>{{text}}</p>
    </div>
</body>

<script type="text/javascript">
    Vue.config.productionTip = false;
    new Vue({
        data() {
            return {
                text: '中午吃什么',
            };
        },
    }).$mount("#app");
</script>

# v-once

  • 只渲染元素和组件一次。随后的重新渲染，元素/组件及其所有的子节点将被视为静态内容并跳过。
  • 不需要表达式
  <body>
    <div id="app">
      <!-- 只渲染一次，后续数据有变更时不再渲染 -->
      <p v-cloak v-once>{{text}}</p>
      <p v-cloak>{{text}}</p>
      <button @click="++text">更新数据</button>
    </div>
  </body>

<script type="text/javascript">
    Vue.config.productionTip = false;
    new Vue({
        data() {
            return {
                text: 1,
            };
        },
    }).$mount("#app");
</script>

# 自定义指令（不重要）

全局自定义指令：

  • 必须在实例化Vue之前创建
  • 使用时：v-指令名
<body>
  <div id="app">
    <input type="text" v-focus />
  </div>
</body>

<script type="text/javascript">
  Vue.directive("focus", {
    inserted(el) {
      // 指令所在的dom
      console.log(el);
      el.focus();
    },
  });
  new Vue({}).$mount("#app");
</script>

局部自定义指令：

  • 只能在注册的组件内使用
<body>
    <div id="app">
        <input type="text" v-focus />
    </div>
</body>

<script type="text/javascript">
    new Vue({
        directives: {
            focus: {
                inserted(el) {
                    console.log (el);
                    el.focus();
                },
            },
        },
    }).$mount("#app");
</script>

# 数据代理和数据劫持

_data 就是 data 配置项，最终会通过数据代理代理到vm对象身上。

通过this修改vm身上的数据时，会调用该数据的set方法（set和get都在vm身上）修改_data的数据。这一步代理是为了简化对data中属性的读写操作。

通过this修改（读取）vm身上的数据时，_data是不知道的。因此，_data内的数据也进行了代理（这些数据的get和set在_data上），这样当数据发生变化时，Vue就能知道，并且会调用响应的set方法更新_data数据，并使用新数据进行视图更新。这是第二次代理。

两次代理之后，才完成了数据的更新。第一次是没改到数据的，因为数据最终是存在_data里的，需要第二步通过调用_data里的set方法来完成。第一步是数据代理，第二步是数据劫持。

数据代理是为了简化对data中数据的读写操作，数据劫持是为了监视_data数据变化方便进行视图更新。

1.vue 数据代理

  • 目的: 方便程序员写代码，直接在视图中使用数据
  • 原理： Object.defineProperty();
  • 方法: proxyGetter proxySetter
  • 过程： 当要读取vm上的数据时，就是自动调用 对应的 proxyGetter 到 _data 中去读取数据。当要更新vm上的数据时，就是自动调用 对应的 proxySetter， 到 _data 中去更新数据。

2.vue 数据劫持

  • 目的: 监听 _data 中的数据的变化，然后重新渲染视图。
  • 原理： Object.defineProperty();
  • 方法: reactiveGetter reactiveSetter
  • 过程： 当要读取_data 上的数据时，就是自动调用 对应的 reactiveGetter。当要更新 _data 上的数据时，就是自动调用对应的 reactiveSetter ,去重新渲染视图。

# 数组的响应式

Vue中，数组中的数据没有使用 Object.defineProperty 添加get和set，可以更新数据，但是Vue监听不到，无法更新视图（不是响应式的）。

Vue重新封装了7个方法，使用这些方法操作数组数据时，可以触发视图更新。

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()

TIP

直接改数组本身是可以的
